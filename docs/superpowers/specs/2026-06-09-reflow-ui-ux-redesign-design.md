# Reflow UI/UX 改版设计

日期：2026-06-09
状态：已与用户逐节确认

## 背景与目标

MVP 的界面是功能验证级的：约 170 行模板、12 行 CSS。用户反馈两个具体痛点——①「应用修正」后表单内容残留，②工作区的修改项无法撤销——以及整体评价「不美观、交互不成熟」。

经审视共识别 13 个交互问题（详见「问题清单」）。本次改版目标：**整体升级所有页面的视觉与交互**，系统性修复这些问题。

## 决策摘要

| 决策点 | 结论 |
|---|---|
| 范围 | 整体改版（非仅修两个痛点） |
| 技术路线 | **渐进增强**：保留 Jinja2 + HTMX 服务端渲染，自定义 CSS + 少量 Alpine.js；不引入 Vue/React、不引入构建工具链 |
| 视觉风格 | **工程务实风（GitHub/Primer 式）**：紧凑、灰白底、蓝绿点缀，自定义 CSS 实现，不依赖大框架 |
| 撤销语义 | **仅工作区草稿可撤销**（删 changeset 行，复用 `models.delete_change`）；不记审计日志；已提交节点维持「修正 + 传播/冲突」流程不变 |
| 冲突弹窗取消 | **取消 ≡ 全部「保留下游值」**：修正已记录、下游不动，与现有后端行为一致，无需回滚机制 |
| 新建入口 | **首页唯一「＋新建单板」入口**，单板名称/PCB版本/BOM版本在流程中用可输入下拉框选择或隐式创建；「新建 BOM 版本」不再是独立功能 |

## 问题清单（13 项，全部在本次改版内解决）

节点详情页：1) 无「本节点修改」汇总视图；2) 编辑全靠手输位号且后端不校验（modify 不存在/add 已存在的位号被静默接受）；3) op 与 Part 输入框无联动；4) 操作无反馈；5) 草稿编辑与历史修正界面无区分；6) 冲突确认替换表格且不可取消；7) BOM 表无搜索筛选。
导入：8) 校验有问题仍可导入；9) 成功后无提示、不跳转目标。
其他页面：10) 首页层级感弱、空状态空白；11) 状态图无改动摘要；12) 日志页节点不可读、无筛选。
全局：13) 不存在的资源直接抛异常、表单错误返回裸 JSON。

## 设计详情

### 全局

- `base.html` 统一顶部导航：左侧 Reflow logo（回首页）+ 面包屑（单板层级 / 节点），右侧上下文链接（状态图、审计日志）。页面 `<title>` 随内容变化。
- 自定义 `style.css` 重写为 Primer 风格的设计变量与组件类（徽章、卡片、按钮、表格、提示条、弹窗、toast）。不引入 CSS 框架。
- 引入 **Alpine.js**（CDN，与 HTMX 并列）处理纯客户端交互：表单清空、op/Part 联动、表格筛选、弹窗开合。
- **toast 组件**：所有写操作成功后右下角短暂提示（HTMX 响应触发，自动消失）。
- **错误处理**：访问不存在的单板/节点返回中文 404 页；表单校验失败返回红色提示条（HTMX 场景返回错误片段，整页场景在表单旁渲染），不再出现裸 JSON 422 / 异常白屏。

### 节点详情页（核心页面，两栏布局）

**左栏 = BOM 表**：

- 工具栏：文本筛选框（按位号/Part 过滤，Alpine 客户端过滤）+ 筛选标签「全部 N ／ 本节点修改 n ／ 不贴 m」。
- 状态徽章：「本节点·修改/新增/不贴」（彩色）与「上游·不贴」（灰色）区分修改来源；行底色沿用 add/modify/remove 三色。
- 修改行显示新旧值对比（旧值划线灰显）。
- 行尾快捷操作（悬停显示）：普通行「修改 / 不贴」——点击把位号自动填入右栏表单并选好 op；本节点行「↩ 撤销」；上游不贴行「恢复贴装」（把位号和 op=新增 填入右栏表单，Part 值由用户填写后提交）。

**右栏 = 操作栏**（从上到下）：

1. **「添加修改」表单**：位号输入带 datalist 自动补全；op 三选一按钮（修改/新增/不贴）；选「不贴」时 Part 输入禁用；提交成功后**自动清空表单** + toast「✓ 已应用：R5 修改 → 10kΩ」。
2. **「本节点修改（n）」面板**：当前节点 changeset 列表，草稿节点每行带「↩ 撤销」。
3. **commit 框**：说明输入 + 「提交为新节点」（仅草稿节点显示）。

**后端校验**（新增，纯逻辑层）：`modify`/`remove` 要求位号存在于折叠后 BOM；`add` 要求位号不存在；`modify`/`add` 要求 Part 非空。违反返回中文错误提示。

**历史节点变体**（已提交节点）：顶部黄色警示条「这是已提交的历史节点，编辑＝修正历史记录，会自动向下游传播…」；编辑表单默认收起，点「修正历史记录…」展开；无撤销按钮、无 commit 框。

**冲突确认**：从「替换 BOM 表」改为**弹窗**。逐位号单选「保留下游值 / 采用修正值」，底部「取消」与「确认」。取消＝关闭弹窗，等同全部保留下游值（修正已记录，下游不动）。

### 撤销（痛点 2）

- 新路由：撤销草稿节点某位号的修改。仅当节点 `is_committed=0` 时允许，否则 4xx 中文错误。
- 实现：调 `models.delete_change`；折叠引擎自动恢复继承值；不写审计日志（草稿未提交，不属于正式记录）。
- 入口：右栏「本节点修改」面板每行 + BOM 表「本节点」行尾。

### 首页

- 右上角唯一入口「＋ 新建单板」。版本卡片上不再有任何新建入口，层级列表只做浏览与跳转。
- 按「单板名称 → (PCB版本, BOM版本) → 单板」分组：名称为分组标题，每个版本一张卡片（显示位号数、单板数），单板呈现为可点击芯片（显示节点数；工作区有未提交修改时高亮提示）。
- 空数据库显示引导文案。

### 统一「新建单板」流程（替代原「新建 BOM 版本」页）

- 表单字段：单板名称、PCB版本、BOM版本（三个**可输入下拉框**：datalist 提供已有值，可输入新值）+ 单板 ID。
- 三级组合指向**已有 BOM 版本** → 蓝色提示「已有版本（N 个位号），直接使用其初始 BOM」，无需 CSV。
- 组合是**新 BOM 版本** → 黄色提示并展开 CSV 上传区；上传后内联显示校验结果（问题红色列出；通过则显示位号数和可展开预览）。**校验有问题时提交按钮禁用**。
- 提交：隐式创建层级 + 初始 BOM（如为新版本）+ 单板，跳转到新单板状态图页并 toast。
- 旧路由 `/bom-version/new`、`/bom-version/import-preview`、`POST /bom-version`、`POST /board` 整合为新流程的路由；`csv_import.py` 纯逻辑不变。

### 状态图

- 时间线**最新在上**：工作区草稿置顶（虚线边框 + 「n 条未提交」徽章），向下依次为已提交节点，底部为「初始状态」节点（紫色标识，显示初始 BOM 位号数与导入时间）。
- 每个节点卡片：说明、改动条数徽章、位号摘要（如「R5 修改 · C2 不贴」，过多时截断）、时间。整卡可点击进入节点详情。

### 审计日志

- 节点列显示「#9 试装验证」（id + 说明），可点击跳转。
- 变更列「旧值（划线）→ 新值」直观对比；不贴显示为「→ 不贴」。
- 来源徽章：「直接修改」（蓝）/「上游传播」（紫）。
- 顶部筛选：按位号文本筛选 + 按节点下拉筛选。
- 排序改为最新在上。

## 后端改动清单

| 改动 | 位置 |
|---|---|
| 位号编辑校验（modify/remove 需存在、add 需不存在、Part 非空） | 纯逻辑（新函数，bom_engine 或 propagation 旁） |
| 撤销草稿修改路由（校验 is_committed=0，调 delete_change） | `routes/board.py` |
| 统一新建单板路由（含版本存在性查询、CSV 校验、隐式创建） | `routes/hierarchy.py` 重构 |
| 节点改动摘要查询（状态图用：每节点 changeset 条数与位号列表） | `models.py` |
| 日志查询带节点说明、支持筛选、倒序 | `routes/log.py` |
| 404 处理（get_board/get_node 为 None 时） | 各路由 |
| HTMX 错误片段 / toast 触发约定 | 路由 + 模板 |

传播/冲突核心算法（`propagation.py`）**不改**；冲突取消语义无需后端改动。

## 测试策略

- 遵循项目 TDD 约定：先写失败测试再实现。
- 纯逻辑新增重点测：位号编辑校验规则的各分支。
- 路由测：撤销（草稿可撤、已提交节点拒绝、撤销后折叠值恢复继承）；统一新建流程（已有版本不需 CSV、新版本校验失败禁止创建、隐式创建正确）；404 与表单错误响应。
- 现有 36 个测试应全部保持通过（路由签名变化处同步调整，如 /bom-version 系列）。
- 模板/交互层（Alpine 行为、toast）以手工验证为主，不写浏览器自动化测试。

## 非目标（边界不变）

飞线等位号以外的修改类型、状态回放界面、多用户/并发/权限、工具内测试记录，仍在 MVP 边界外。本次也不做：删除单板/版本、暗色主题、移动端适配。
