# Header 三点菜单收纳 设计文档

日期：2026-07-18
状态：已确认

## 背景与目标

header（`.topnav`）在移动端和桌面端都过于臃肿：各页面的功能链接/按钮（`ctxlinks` 块）、哈希跳转框、主题切换按钮挤在一起，窄屏下换行堆成四五行。本次把**所有功能入口收进一个三点（⋯）弹出菜单**，桌面与移动共用同一套结构。

本设计**取代** `2026-07-16-mobile-adaptation-design.md` 中「细节设计 §1 Header」一节的换行堆叠方案。

## 方案（已选定）

功能区**永远**住在弹出菜单里，不做桌面/移动双形态 DOM。曾考虑的备选：

- `display:contents` 包裹层，桌面行内展开、窄屏变弹层——用户明确桌面也要收纳，不再需要双形态。
- 菜单里 `{{ self.ctxlinks() }}` 重复渲染一份——DOM/Alpine 状态/`data-testid` 重复，弃用。

## 细节设计

### DOM（`base.html`，各页面 `ctxlinks` 块零改动）

header 子元素顺序：`brand` → `.crumbs` → `.hash-jump` → 三点按钮 → 弹出面板 `.topnav-actions`（内含 `{% block ctxlinks %}` + 「切换夜间模式」菜单项）。

Alpine：header 挂 `x-data="{nav:false}"`；⋯ 按钮切换；`@click.outside` 关闭；点击面板内任意项后关闭（面板容器上委托 `@click="nav=false"`，主题切换项同样适用，切完即关，不做特殊处理）。

### 布局

- **桌面（>720px）**：单行——logo | 面包屑 | （弹性空隙） | 哈希跳转框 | ⋯。
- **窄屏（≤720px）**：三行——logo + ⋯（首行两端）/ 面包屑（整行，可换行）/ 哈希跳转框（整行）。

### 弹出面板样式

- 复用 `.menu-pop` 观感：`var(--surface)`、`var(--border)`、`--menu-shadow`、圆角，两套主题天然支持；绝对定位在 ⋯ 下方右对齐，`z-index` 高于页面内容。
- 面板内的 `a`、`button`（含 `.btn`、`.btn-outline`、`.btn-link`）统一覆盖成纵向菜单项：全宽、左对齐、去边框底色、hover 用 `var(--surface-2)`；危险项保持 `var(--red)`；图标/emoji 前缀原样保留。
- ⋯ 按钮复用 `.menu-btn` 观感，触屏点按区加大（既有 `@media (hover:none)` 规则受益）。

### 交互保留

菜单里的 HTMX 按钮（删除节点、添加到草稿）、Alpine 对比切换（`@toggle-cmp.window`）、普通链接行为不变，只是换容器；点击项先关菜单，`hx-confirm` 确认框随后弹出，互不干扰。无 `ctxlinks` 的页面菜单只剩「切换夜间模式」一项，仍然有意义。

### 取舍（已确认）

「＋ 新建单板」「对比节点」等高频入口也全收进菜单，多一次点击。若日后觉得某入口太深，再单独捞出，不在本次范围。

## 测试

- 现有 UI 测试若直接点击 `ctxlinks` 里的按钮/链接，需先点开 ⋯ 菜单——跑基线（~491 passed），失败用例补「先点 ⋯」步骤。
- 完成后按风格指南自检：两套主题 × 桌面/窄屏实机目检。
