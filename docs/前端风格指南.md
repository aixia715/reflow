# Reflow 前端风格指南

> **改任何前端文件（`app/templates/`、`app/static/`）之前必读。**
> 目标：风格一致、两套主题都好看、交互符合本工具已有的习惯。改完务必对照文末自检清单。

## 设计原则

- 视觉基调对齐 GitHub：浅灰底、白色卡片、细边框、小圆角、13–14px 字号。克制，不花哨。
- 信息密度优先：这是工程师每天用的工具，不是营销页。少留白堆叠，多用紧凑的 panel/badge/chip。
- 桌面单栏布局，`main` 最大宽度 1100px，居中。不做响应式断点（单人桌面工具）。

## 设计令牌（最重要的一条规则）

所有颜色**只允许用 `style.css` 顶部 `:root` 里的 CSS 变量**，禁止在模板或新样式里写裸色值（`#xxx`、`rgb()`）。唯一例外是 `rgba()` 阴影和半透明边框（如 `#cf222e66`）。

- 夜间模式实现方式：`[data-theme="dark"]` 块**只覆盖颜色变量**，不写任何结构样式。
- **新增一个颜色变量，必须同时在 `[data-theme="dark"]` 块里给出夜间值**，否则夜间模式会破。
- 语义变量速查：
  - 底色/文字：`--bg` 页面底、`--surface` 卡片、`--surface-2` 次级面、`--fg` 正文、`--muted` 次要文字
  - 边框：`--border` 常规、`--border-soft` 表格行/分隔线
  - 语义色：蓝=链接/选中/进行中，绿=主操作/新增，红=危险/删除，黄=警告/修改，紫=根节点/特殊
  - 每个语义色有前景（`--blue`）和浅底（`--blue-bg`）成对出现，badge/flash/高亮行成对使用
- 圆角统一 `var(--radius)`（6px），药丸形（badge/chip）用 `999px`。

## 组件清单（先复用，再新建）

写新界面前先翻 `style.css`，下面这些已有组件覆盖了绝大多数需求：

| 类 | 用途 |
|---|---|
| `.panel` / `.panel-title` | 白底卡片容器；侧栏、表单、分组都用它 |
| `.badge` `.badge-{blue,green,yellow,red,purple}` | 状态标签（已提交/未提交/数量） |
| `.chip` / `.chip.pending` | 可点击的药丸链接（单板入口） |
| `.btn` / `.btn-primary`（绿实心，每页至多一个）/ `.btn-outline`（`.danger` 红边框+红字）/ `.btn-link`（`.danger` 红） | 按钮层级；非表单元素（如 `<a class="btn">`）用 `.disabled` 类做伪禁用态（`opacity:.5` + `pointer-events:none`），配合 `aria-disabled`/`tabindex="-1"` |
| `.input` / `label` | 表单控件，13px |
| `.flash` `.flash-{error,warn,info}` | 页面内提示条 |
| `.toast`（经 `showToast()`） | 操作成功的瞬时反馈 |
| `.modal-overlay` / `.modal` / `.modal-head` / `.modal-foot` | 弹窗（目前仅冲突确认） |
| `.empty` | 虚线框空状态，必须带引导文案 |
| `.icon-btn`（`.danger` 红 / `.hover-only` 配合表格） | 内联 SVG 图标按钮（笔=回填编辑、撤销=箭头）：色随 `currentColor`、默认 `--muted`，hover 变蓝、`.danger` 变红；务必带 `title`+`aria-label` |
| `.ico` / `.row-icons` | 15px 线性图标本体（`stroke:currentColor`）/ 行内图标按钮组横向容器 |
| `.svg-sprite` | `base.html` 顶部承载 `<symbol>` 的隐藏 SVG sprite 容器（`position:absolute;w/h:0`） |
| `table.bom` + `tr.row-{add,modify,remove}` | BOM 表格及差异行底色 |
| `.timeline` / `.tl-item` / `.tl-card` | 状态图节点链；对比模式下硬更改用 `.tl-item.hard.disabled` 置灰且禁用交互 |
| `.seg` | 分段切换器（筛选 tab） |
| `.cmp-picker` | 对比页「板 + 节点」下拉选择器组的排版容器（内部复用 `.input`） |
| `.hash-jump` | header 里的紧凑哈希跳转输入框排版容器（内部复用 `.input`；提交走 `/hash-lookup`，命中整页跳转、出错弹 toast） |
| `.two-col` | 主内容 + 300px 右侧栏布局 |
| `.page-head` / `.muted` / `.ml-auto` | 页头、次要文字、右推 |

新建组件的条件：现有组件确实不能组合出来。新类写进 `style.css` 对应区块附近（文件按「全局 → 组件 → 页面特有 → 弹窗/toast → 夜间」分区），命名跟随现有风格（短横线小写、语义命名）。**不引入任何 CSS 框架、不加构建步骤、不写内联 style。**

## 页面骨架

- 一律 `{% extends "base.html" %}`，填四个 block：`title`（格式「页面 — Reflow」）、`crumbs`（面包屑，首页开头）、`ctxlinks`（右侧上下文链接/主操作按钮）、`content`。
- 局部模板（HTMX swap 用）以 `_` 开头，不 extends。
- 页面顶部用 `.page-head`：`h1`（18px）+ 状态 badge + 右侧 `.stable-link` 等。

## 交互模式（HTMX + Alpine 惯用法）

整体约定见 CLAUDE.md「前端约定」一节，此处是落地细节：

- **局部刷新优先**：编辑、撤销、删除走 HTMX；只有创建/跳转类操作整页跳转（用 `?flash=` 带 toast 文案）。
- **校验失败**：后端返回 200 + `HX-Retarget: #form-error`，错误就地显示在表单旁，不弹窗、不跳页。
- **操作成功**：`HX-Trigger: {"showToast": "…"}` 给瞬时反馈；toast 3.5s 自动消失，不需要用户点掉。
- **危险操作**三件套，缺一不可：
  1. 入口低调——`.icon-btn.danger`（图标）或红色 `.btn-link.danger`，绝不用大红实心按钮常驻页面；
  2. `hx-confirm` 确认，文案格式固定：`确认删除「对象名」？此操作将…，不可恢复。`（写明波及范围）；
  3. 完成后 toast 或 flash 告知结果。
- **悬停渐显**：表格行内操作按钮加 `.hover-only`，避免视觉噪音。
- Alpine 监听 htmx 事件加 `.camel` 修饰符；模板向 `hx-vals`/JS 传值一律 `|tojson` 且 HTML 属性用单引号。
- 弹窗挂载点是 base.html 的 `#modal`；取消必须等价于「什么都不做」的安全选项。
- **弹出菜单的关闭机制**：菜单按钮若带 `@click.stop` 或 `stopPropagation()`（阻止冒泡），就**不能**用 `@click.outside` 关闭——它依赖事件冒泡，会被 stop 拦截，导致点别处时菜单不消失。这种情况用 `@close-menus.window="if (!$el.contains($event.detail)) open=false"`：base.html 在**捕获阶段**对任意点击广播 `close-menus`（先于按钮的 stop 执行），菜单据此判断点击是否落在自身之外来决定关闭（见 `state_graph.html`）。按钮不带 stop 的简单菜单仍可用 `@click.outside`。

## 文案（人性化的主要载体）

- 全部中文，语气直接、信息完整：说清**发生了什么、影响范围、下一步**。
  - 好：「⚠ 这是已提交的历史节点。在此编辑＝**修正历史记录**，会自动向下游传播。」
  - 差：「警告：操作有风险。」
- 空状态必须给出第一步引导（参考首页 `.empty`：「点右上角『＋ 新建单板』上传初始 BOM CSV 开始。」）。
- 图标按钮必须有 `title` 提示。
- 数字信息用 `·` 分隔的紧凑写法：`12 个位号 · 3 块单板`。
- 错误消息也是中文（前后端一致）。
